# Unitree G1 MuJoCo 与 ROS 2 导航仿真技术说明

## 1. 项目目标

本项目面向 Unitree G1 双足机器人，在 MuJoCo 与 ROS 2 中建立完整的导航仿真
闭环，主要实现：

1. MuJoCo 中的 G1 接触动力学仿真；
2. 基于强化学习策略的双足行走；
3. 二维激光雷达模拟；
4. ROS 2 在线建图；
5. 全局路径规划；
6. 局部动态避障；
7. RViz 地图、膨胀层、激光、路径和机器人状态可视化；
8. 为后续导航算法真机迁移保留接口。

系统中的机器人通过腿部关节运动产生真实位移，不直接修改浮动基座位置，避免
出现机器人在地面漂移的问题。

## 2. 系统总体架构

导航系统的数据链路如下：

```text
RViz / ROS 2 导航目标
          ↓
全局路径规划器
          ↓
局部路径跟踪与动态避障
          ↓
机器人速度指令
          ↓
G1 强化学习行走策略
          ↓
腿部关节目标与 PD 力矩控制
          ↓
MuJoCo 接触动力学
          ↓
机器人实际位姿、关节状态和模拟激光
          ↓
ROS 2 SLAM、地图更新与路径重规划
```

系统将视觉环境、物理碰撞、激光感知和导航规划分层处理：

| 层级 | 作用 |
|---|---|
| MuJoCo 视觉层 | 显示机器人和办公室环境 |
| MuJoCo 物理层 | 处理机器人接触、地面摩擦和障碍物碰撞 |
| 激光感知层 | 向 ROS 2 发布静态和动态障碍物距离 |
| SLAM 地图层 | 在线生成二维占据栅格地图 |
| 全局规划层 | 在障碍物与膨胀层之外规划路径 |
| 局部控制层 | 跟踪路径并实时躲避动态障碍物 |

## 3. MuJoCo 与 Unitree G1 动力学

MuJoCo 场景加载宇树 G1 官方 12 自由度强化学习行走模型。系统没有使用直接积分
位置的方式移动机器人，而是采用真实动力学控制链路：

```text
导航速度指令
   ↓
G1 强化学习行走策略（50 Hz）
   ↓
12 个腿部关节目标
   ↓
PD 力矩控制
   ↓
MuJoCo 接触动力学（500 Hz）
   ↓
机器人实际位置、姿态和速度
```

强化学习策略输入包括：

- 机器人机身角速度；
- 重力在机身坐标系中的方向；
- 导航线速度与角速度指令；
- 腿部关节位置；
- 腿部关节速度；
- 上一时刻动作；
- 步态相位。

策略输出 12 个腿部关节动作。动作经过比例缩放后转换为关节目标角度，再由
PD 控制器计算关节力矩：

```text
τ = Kp × (q_target - q) - Kd × dq
```

MuJoCo 根据关节力矩、机器人质量、地面摩擦和接触约束计算机器人实际运动。
因此仿真中的 G1 通过交替迈步前进、侧移和转向，运动逻辑与真实双足机器人一致。

## 4. ROS 2 导航节点

系统包含以下核心节点：

| 节点 | 作用 |
|---|---|
| `g1_rl_mujoco_navigation.py` | G1 动力学、行走策略、模拟激光、里程计和动态障碍物 |
| `slam_toolbox` | 使用静态激光数据执行在线二维 SLAM |
| `dijkstra_planner` | 在膨胀地图上生成全局安全路径 |
| `nav2point` | 路径跟踪、速度控制和局部动态避障 |
| `robot_state_publisher` | 发布 G1 机器人模型的 TF 关系 |
| `rviz2` | 显示地图、膨胀层、激光、路径、目标和机器人 |

导航相关启动文件：

```text
launch/navigation_mapping.launch.py
```

## 5. ROS 2 话题与 TF

主要 ROS 2 话题如下：

| 话题 | 消息内容 |
|---|---|
| `/scan_static` | 只包含静态场景的模拟激光，用于 SLAM |
| `/scan` | 包含静态和动态障碍物的模拟激光，用于局部避障 |
| `/lidar_odometry/pose_fixed` | MuJoCo 中机器人的实际里程计 |
| `/joint_states` | G1 关节位置、速度和力矩 |
| `/map` | `slam_toolbox` 生成的在线占据栅格地图 |
| `/map_inflated` | 全局规划器生成的障碍物膨胀地图 |
| `/g1pilot/goal` | 用户设置的导航目标 |
| `/g1pilot/path` | 全局规划路径 |
| `/g1pilot/auto_joy` | 局部控制器输出的机器人速度指令 |
| `/g1pilot/auto_enable` | 自动导航使能状态 |

主要 TF 关系如下：

```text
map
└── odom
    └── base_footprint
        ├── base_link
        │   └── pelvis
        └── base_scan
```

其中：

- `base_footprint` 表示机器人在地面上的平面位姿；
- `base_link` 保留 MuJoCo 中实际机身高度、横滚和俯仰；
- `pelvis` 用于连接 G1 机器人关节模型；
- `base_scan` 是模拟激光雷达坐标系。

## 6. 模拟激光与地图分层

MuJoCo 节点模拟 360° 二维激光雷达，并分别发布两个话题。

### 6.1 静态激光 `/scan_static`

该话题只包含墙体、家具等静态障碍物，供 `slam_toolbox` 建图使用。

使用静态激光建图可以避免移动障碍物长期残留在地图中，防止形成错误的“幽灵
障碍”。

### 6.2 实时激光 `/scan`

该话题同时包含：

- 静态墙体和家具；
- 移动箱体；
- 移动行人模型。

`nav2point` 使用该话题进行实时局部避障，RViz 也通过该话题显示当前障碍物。

模拟激光主要参数：

```text
扫描范围：360°
角分辨率：1°
最小距离：0.10 m
最大距离：8.0 m
目标更新周期：约 0.04 s
```

## 7. ROS 2 在线建图

系统使用 `slam_toolbox` 的在线异步模式生成二维占据栅格地图。

输入数据包括：

- `/scan_static` 静态激光；
- `odom → base_footprint` 机器人里程计与 TF。

输出包括：

- `/map` 在线占据地图；
- `map → odom` SLAM 坐标变换。

地图中的主要栅格状态为：

| 数值 | 含义 |
|---:|---|
| `-1` | 未知区域 |
| `0` 附近 | 可通行区域 |
| `50` 以上 | 障碍区域 |

机器人行走过程中，激光扫描和里程计持续更新，RViz 中的地图会同步扩展。

## 8. 全局路径规划

系统使用自定义 Dijkstra 规划器在二维占据地图上搜索全局路径。

### 8.1 地图膨胀

规划器在原始地图基础上生成 `/map_inflated`。当前障碍物膨胀半径为：

```text
0.40 m
```

膨胀层综合考虑 G1 机身宽度、步态摆动和定位误差，使机器人中心不会贴近墙面或
家具边缘运行。

### 8.2 路径安全约束

全局规划器执行以下约束：

- 默认禁止进入未知区域；
- 将占据值不低于 50 的栅格视为障碍物；
- 禁止路径进入障碍物膨胀层；
- 禁止斜向穿过两个相邻障碍物形成的墙角；
- 起点或目标位于障碍物时拒绝规划；
- 起点或目标位于膨胀层时拒绝规划；
- 目标位于未知区域或地图外时拒绝规划；
- 地图更新时按照 1 Hz 频率重新规划。

### 8.3 路径优化

Dijkstra 搜索完成后，系统依次执行：

1. 删除距离过近的冗余路径点；
2. 在膨胀地图上进行安全路径快捷化；
3. 使用 Catmull-Rom 曲线尝试平滑路径；
4. 对平滑结果逐段进行碰撞检查；
5. 平滑路径不安全时退回未平滑安全路径。

最终发布的路径必须通过膨胀地图碰撞验证，不会使用穿过障碍物或膨胀层的直线
作为降级路径。

## 9. 路径跟踪

`nav2point` 接收 `/g1pilot/path`，根据机器人当前位置选择前方路径点，并生成：

- 前向速度；
- 横向速度；
- 偏航角速度。

主要控制参数：

```text
前向速度上限：0.60 m/s
横向速度上限：0.60 m/s
角速度上限：0.50 rad/s
路径点容差：0.20 m
目标容差：0.10 m
```

速度指令通过 `/g1pilot/auto_joy` 发送给 MuJoCo G1 行走控制器，再由强化学习
策略转换为腿部关节动作。

## 10. 局部动态避障

局部避障直接使用 `/scan` 的实时激光数据，不依赖下一次全局地图更新。

当前主要参数：

```text
避障作用半径：1.20 m
紧急停止距离：0.45 m
斥力增益：0.85
```

局部避障处理逻辑：

1. 统计机器人周围障碍物距离；
2. 对 1.20 m 范围内的障碍物计算排斥速度；
3. 障碍物接近前方时降低前进速度；
4. 距离小于 0.45 m 时停止继续前进；
5. 比较左右两侧可用空间；
6. 向空间较大的一侧侧移并转向；
7. 绕过障碍物后重新跟踪全局路径。

全局规划负责长期静态路径，局部避障负责短时间动态变化，两者共同构成分层导航
系统。

## 11. 动态障碍物测试

MuJoCo 场景包含两类动态障碍物：

| 障碍物 | 形状 | 运动方式 |
|---|---|---|
| `dynamic_box` | 移动箱体 | 周期性横向移动 |
| `dynamic_pedestrian` | 圆柱行人模型 | 在场景中往返移动 |

动态障碍物具有 MuJoCo 物理碰撞几何，同时参与 `/scan` 的距离计算，但不进入
`/scan_static` 和长期 SLAM 地图。

该设计用于测试机器人在全局路径保持不变时，能否实时减速、侧移、转向和重新
回到路径。

## 12. RViz 可视化

RViz 配置文件：

```text
config/navigation_mapping.rviz
```

主要显示项：

| 显示项 | 作用 |
|---|---|
| `Online SLAM Map` | 显示 `/map` 在线占据地图 |
| `Inflated Costmap` | 显示 `/map_inflated` 障碍物膨胀层 |
| `Simulated LiDAR` | 显示 `/scan` 实时激光点 |
| `Planned Path` | 显示 `/g1pilot/path` 全局路径 |
| `Unitree G1` | 显示机器人模型和关节运动 |
| `Goal` | 显示最终导航目标 |
| `Current Waypoint` | 显示当前跟踪路径点 |

用户可以使用 RViz 工具栏中的 `2D Goal Pose` 在已知可通行区域设置目标点。

设置目标后，应重点观察：

- 目标是否位于已知自由区域；
- 绿色路径是否绕开障碍物和膨胀层；
- 红色激光点是否包含动态障碍物；
- 机器人是否通过迈步沿路径移动；
- 动态障碍物接近时机器人是否减速或绕行；
- 地图变化后全局路径是否重新规划。

## 16. 常见问题

### 16.1 全局路径为空

可能原因：

- 目标位于未知区域；
- 目标位于地图范围外；
- 目标位于障碍物或膨胀层；
- 起点周围没有足够净空；
- 已知自由区域与目标不连通。

处理方法：

- 等待 SLAM 地图完成当前区域更新；
- 在 RViz 白色自由区域内重新设置目标；
- 避开膨胀层；
- 检查 `/g1pilot/path` 和规划器日志。

### 16.2 机器人不移动

检查：

- `/g1pilot/path` 是否包含路径点；
- `/g1pilot/auto_enable` 是否为 `true`；
- `/g1pilot/auto_joy` 是否持续发布；
- MuJoCo 窗口是否仍在运行；
- G1 是否因碰撞或失稳倒地。

### 16.3 RViz 没有地图

检查：

- `/scan_static` 是否持续发布；
- `odom → base_footprint` TF 是否存在；
- `slam_toolbox` 是否启动；
- RViz Fixed Frame 是否设置为 `map`。

### 16.4 路径穿过膨胀层

正常情况下规划器会拒绝这类路径。如出现异常，应检查：

- RViz 显示的是否为 `/map_inflated`；
- 运行的代码是否为最新构建版本；
- 是否重新执行过 `colcon build --symlink-install`；
- 路径消息是否来自当前 `dijkstra_planner`。

## 17. 真机迁移注意事项

仿真导航迁移到真实 G1 前，需要完成：

1. 将模拟 `/scan` 替换为真实激光雷达数据；
2. 标定雷达、机身和足底坐标系；
3. 校准真实机器人里程计或激光定位；
4. 根据真实 G1 外形重新确认膨胀半径；
5. 标定真实前进、侧移和转向速度上限；
6. 对 RL 行走策略执行 Sim2Real 验证；
7. 增加急停、通信超时和控制失效保护；
8. 在封闭区域以低速逐步测试；
9. 验证动态行人避障和紧急制动距离；
10. 在安全员监护下再进入真实办公室运行。

仿真中的路径规划和局部避障逻辑可以复用，但传感器噪声、定位误差、地面摩擦、
网络延迟和行人行为必须按照真机环境重新测试。

## 18. 当前结论

项目已经实现 Unitree G1 在 MuJoCo 中的双足动力学导航，并打通 ROS 2 在线建图、
障碍物膨胀、全局安全路径规划、局部动态避障和 RViz 可视化。

机器人通过强化学习行走策略和腿部关节力矩完成移动，不依赖基座漂移。全局规划
路径经过膨胀地图碰撞校验，局部控制器能够使用实时激光对动态障碍物执行减速、
停止、侧移和绕行。

当前系统可用于导航算法联调、动态避障测试、路径安全验证和后续真机迁移准备。
